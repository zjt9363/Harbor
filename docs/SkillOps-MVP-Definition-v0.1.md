# SkillOps MVP 定义 v0.1

## 1. 项目目标

SkillOps 是一个面向团队内部使用的本地化 Agent 平台。第一阶段目标是搭建一个可在局域网访问的 Web 系统，让团队成员能够通过统一聊天界面与本地部署的开源 Agent 交互，并完成基础的文件上传、对话和结果查看。

## 2. 目标用户

第一版面向团队内部成员使用，包括但不限于：

- 客户端开发
- 工具开发
- 技术支持人员
- 后续可能接入性能分析或验证流程的工程成员

MVP 阶段默认用户规模较小，以内部试用和验证为主。

## 3. MVP 要解决的问题

第一版只解决最基础的问题：

- 团队成员可以通过浏览器访问一个统一入口
- 用户可以在 Web 页面中与本地 Agent 对话
- 用户可以上传文本文件和图片作为对话输入
- Agent 可以返回文本回复，并在页面中展示
- 系统可以保存基础的历史会话和消息记录

也就是说，MVP 的目标不是“做一个完整智能平台”，而是先跑通一条稳定的使用主链路。

## 4. MVP 范围

第一版包含以下能力：

- Web 聊天界面
- 用户输入文本并发送消息
- 上传文件和图片
- 展示 Agent 回复
- 保存历史会话
- 基础的局域网访问能力
- 接入一个本地部署的开源模型
- 通过 Agent Runtime 转发请求并生成回复

## 5. MVP 明确不做的内容

为了保证项目可以快速落地，以下内容不纳入第一版范围：

- 不做复杂多 Agent 协作
- 不做完整 Skill 编排系统
- 不做自动流水线执行
- 不做 UE Trace 专项解析能力
- 不做复杂权限体系和多租户设计
- 不做高级知识库 / RAG 系统
- 不做移动端或桌面端客户端
- 不做高度定制的 Prompt 调优体系
- 不做复杂任务调度与异步作业系统

这些能力后续可以扩展，但不应进入 MVP 的首轮实现。

## 6. 系统主链路

第一版主链路定义为：

1. 用户在浏览器中打开 SkillOps Web 页面
2. 用户输入文本，或上传文件/图片
3. 前端将请求发送到 API Server
4. API Server 调用 Agent Runtime
5. Agent Runtime 调用本地模型服务
6. 模型生成回复并返回
7. 前端展示结果
8. 会话、消息、附件信息被保存

这条链路必须优先跑通，并保持稳定。

## 7. 第一版推荐架构

MVP 阶段采用基础分层架构：

- Browser
- SkillOps Frontend（React Web Client）
- Nginx
- API Server
- OpenClaw Gateway / Agent Runtime
- Local Model Server（如 vLLM 或 Ollama）
- PostgreSQL
- File Storage

架构原则是：

- 先单体集成为主
- 模型层与业务层解耦
- 后续可扩展 skill、任务和分析能力

## 8. 成功标准

当满足以下条件时，可认为 MVP 达标：

- 团队成员可以在局域网内通过浏览器访问系统
- 用户可以成功创建会话并发送消息
- 用户可以上传至少一种文本文件和一种图片文件
- Agent 能稳定返回可展示的文本结果
- 历史会话和消息能被保存并再次查看
- 系统在小范围多人使用下可稳定运行

## 9. 技术原则

第一版遵循以下原则：

- 优先可落地，不追求一次性做全
- 优先打通主链路，不提前扩展复杂能力
- 依赖成熟组件完成 Agent 运行和模型推理
- 业务系统与模型推理解耦
- 架构设计为后续 skill 和分析能力预留接口

## 10. 后续扩展方向

在 MVP 稳定后，后续可以逐步增加：

- Skill 接入与执行框架
- 流水线验证能力
- 性能分析工作流
- Trace / 日志专项解析
- 异步任务系统
- 更细粒度权限控制
- 报告生成与结果导出
- 更强的模型和工具链接入能力

## 11. 一句话版本

SkillOps MVP 的目标不是一开始做成完整的智能工程平台，而是先做出一个团队可访问、可聊天、可上传文件、可接本地 Agent 的基础内网 Web 系统。
